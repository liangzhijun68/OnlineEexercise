好的，这次我把**SQL标准命令**、**SQLite特有SQL命令**和**SQLite点命令**拆成三组，按功能分类对比，让你一眼看清区别：

---

## 一、数据操作命令（DML）

| 功能 | **SQL标准命令** | **SQLite实现/特性** | **对应点命令** |
| :--- | :--- | :--- | :--- |
| 查询数据 | `SELECT ... FROM ...` | 支持`LIMIT`、`OFFSET`分页 | ❌ 无 |
| 插入数据 | `INSERT INTO ... VALUES ...` | 支持`INSERT OR REPLACE`（冲突处理） | ❌ 无 |
| 更新数据 | `UPDATE ... SET ... WHERE ...` | 支持`UPDATE ... FROM`（联表更新） | ❌ 无 |
| 删除数据 | `DELETE FROM ... WHERE ...` | 支持`DELETE ... RETURNING`（返回删除行） | ❌ 无 |

---

## 二、数据定义命令（DDL）

| 功能 | **SQL标准命令** | **SQLite实现/特性** | **对应点命令** |
| :--- | :--- | :--- | :--- |
| 创建表 | `CREATE TABLE ...` | 支持`WITHOUT ROWID`选项 | ❌ 无 |
| 修改表 | `ALTER TABLE ... ADD COLUMN ...` | **仅支持重命名表和添加列**，不支持删除列 | ❌ 无 |
| 删除表 | `DROP TABLE ...` | 标准实现 | ❌ 无 |
| 创建索引 | `CREATE INDEX ... ON ...` | 支持`CREATE INDEX IF NOT EXISTS` | ❌ 无 |
| 删除索引 | `DROP INDEX ...` | 标准实现 | ❌ 无 |
| 创建视图 | `CREATE VIEW ... AS ...` | 支持临时视图 | ❌ 无 |

---

## 三、事务控制命令（TCL）

| 功能 | **SQL标准命令** | **SQLite实现/特性** | **对应点命令** |
| :--- | :--- | :--- | :--- |
| 开始事务 | `BEGIN TRANSACTION` | 支持`BEGIN DEFERRED/IMMEDIATE/EXCLUSIVE`（三种锁模式） | ❌ 无 |
| 提交事务 | `COMMIT` | 标准实现，也支持`END` | ❌ 无 |
| 回滚事务 | `ROLLBACK` | 支持`ROLLBACK TO SAVEPOINT` | ❌ 无 |
| 保存点 | `SAVEPOINT name` | 完全支持嵌套保存点 | ❌ 无 |

---

## 四、SQLite特有SQL命令（非标准）

| 命令/语法 | **功能说明** | **使用示例** |
| :--- | :--- | :--- |
| `INSERT OR REPLACE` | 若主键冲突则替换整行 | `INSERT OR REPLACE INTO t(id,name) VALUES(1,'a');` |
| `INSERT OR IGNORE` | 若冲突则忽略本次插入 | `INSERT OR IGNORE INTO t VALUES(2,'b');` |
| `REPLACE INTO` | 等同于`INSERT OR REPLACE` | `REPLACE INTO t(id,name) VALUES(1,'c');` |
| `PRAGMA` | 设置或查询数据库内部状态 | `PRAGMA table_info(users);`（查看表结构）<br>`PRAGMA foreign_keys=ON;`（开启外键约束） |
| `ATTACH DATABASE` | 附加另一个数据库文件 | `ATTACH 'other.db' AS db2;` |
| `DETACH DATABASE` | 分离已附加的数据库 | `DETACH DATABASE db2;` |
| `VACUUM` | 重建数据库文件，回收碎片空间 | `VACUUM;` 或 `VACUUM INTO 'backup.db';` |
| `ANALYZE` | 收集查询优化器用的统计信息 | `ANALYZE;` 或 `ANALYZE table_name;` |

---

## 五、SQLite点命令（仅命令行工具）

按功能分类列出常用点命令：

### 📋 查看与导航类
| 点命令 | 功能 | 示例 |
| :--- | :--- | :--- |
| `.tables` | 列出所有表（可带通配符） | `.tables user%` |
| `.schema` | 查看表或索引的创建语句 | `.schema users` |
| `.databases` | 显示当前已附加的数据库列表 | `.databases` |
| `.indexes` | 列出指定表的所有索引 | `.indexes users` |
| `.tableinfo` | 显示表的详细列信息（需开启） | `.tableinfo users` |

### 📤 输出格式控制类
| 点命令 | 功能 | 示例 |
| :--- | :--- | :--- |
| `.mode` | 切换输出模式（column/csv/list/insert等） | `.mode column` |
| `.headers` | 开关列头显示 | `.headers ON` |
| `.width` | 设置列宽（column模式） | `.width 20 10 30` |
| `.nullvalue` | 设置NULL值显示为指定字符串 | `.nullvalue (NULL)` |

### 💾 导入导出类
| 点命令 | 功能 | 示例 |
| :--- | :--- | :--- |
| `.dump` | 以SQL语句形式导出整个数据库或表 | `.dump users > backup.sql` |
| `.output` | 将查询结果写入文件而非屏幕 | `.output result.txt` |
| `.import` | 从CSV等文件导入数据 | `.import data.csv mytable` |
| `.read` | 执行SQL脚本文件 | `.read init.sql` |

### ⚙️ 环境与设置类
| 点命令 | 功能 | 示例 |
| :--- | :--- | :--- |
| `.open` | 打开或创建数据库文件 | `.open mydb.db` |
| `.quit` 或 `.exit` | 退出sqlite3命令行 | `.quit` |
| `.help` | 显示所有点命令帮助 | `.help` |
| `.shell` | 执行系统shell命令 | `.shell ls -l` |

## 六、核心术语对比

| 术语 | **SQL标准含义** | **SQLite中的特殊含义/实现** |
| :--- | :--- | :--- |
| **主键** | 唯一标识行的列 | 支持`AUTOINCREMENT`（严格递进），但默认的`ROWID`本身就是自增 |
| **外键** | 引用其他表的约束 | **默认关闭**，需用`PRAGMA foreign_keys=ON;`启用 |
| **事务** | ACID特性的操作单元 | 支持**四种隔离级别**：DEFERRED、IMMEDIATE、EXCLUSIVE、AUTOCOMMIT |
| **索引** | 加速查询的数据结构 | 支持**部分索引**（`WHERE`条件过滤）和**表达式索引** |
| **触发器** | 在事件发生时自动执行的SQL | 支持`BEFORE`/`AFTER`/`INSTEAD OF`，可针对`INSERT`/`UPDATE`/`DELETE` |
| **视图** | 虚拟表 | 支持**临时视图**，但**默认只读**（如需更新需创建`INSTEAD OF`触发器） |
| **存储过程** | 服务器端预编译程序 | ❌ **不支持**（SQLite无服务器进程） |
| **数据类型** | 严格类型约束（INT/VARCHAR等） | **动态类型**，列仅有"类型亲和性"（推荐但非强制） |
