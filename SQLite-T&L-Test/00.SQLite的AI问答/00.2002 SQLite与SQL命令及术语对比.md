要厘清SQLite与SQL命令及术语的关系，核心在于理解：**SQL是标准语言，而SQLite是它的一个具体实现**，就像一个“方言”。同时，SQLite自带了一套独有的“点命令”，用于便捷管理数据库。

### 1. 核心概念区分

*   **SQL（结构化查询语言）**：是操作关系型数据库的**标准语言**。定义了`SELECT`、`INSERT`、`CREATE TABLE`等标准语法。
*   **SQLite**：一个**嵌入式关系型数据库**，它**实现了大部分SQL-92标准**。但它不是独立的服务器进程，而是一个嵌入到应用中的库，将数据库存为单文件。
*   **点命令**：`sqlite3`命令行工具独有的命令（如`.schema`、`.dump`），以点开头，**不属于SQL标准**，用于格式化输出或查看数据库结构。

### 2. SQLite中的SQL方言特性

与MySQL或PostgreSQL等数据库相比，SQLite的SQL语法有自己独特的规则和限制：

*   **动态类型**：列不强制数据类型，而是有“**类型亲和性**”。例如，`INTEGER`列也可存入字符串。
*   **查询限制**：用`LIMIT`子句实现分页，而不是`TOP`关键字；支持`LEFT JOIN`，但不支持`RIGHT JOIN`和`FULL JOIN`。
*   **标识符引用**：建议用双引号`" "`包裹表名或列名，也支持方括号`[ ]`以兼容SQL Server，但后者并非官方推荐的标准做法。
*   **事务特性**：支持`SAVEPOINT`（保存点）实现嵌套事务，且默认每个`INSERT`或`UPDATE`都是自动提交的。

### 3. 核心术语速查

**基础通用术语**：
*   **表、行、列**：数据的基本存储结构。
*   **主键**：唯一标识表中每一行的列。
*   **外键**：用于建立表之间关系的约束。
*   **视图**：基于查询结果的虚拟表。
*   **索引**：用于加速数据检索的数据结构。
*   **事务**：一组要么全部执行、要么全部不执行的数据库操作单元。

**SQLite特有术语**：
*   **类型亲和性**：列对数据类型的倾向性，而非强制约束。
*   **ROWID**：每个表默认自增的隐藏列，可用`SELECT rowid, * FROM table`查看。

### 4. 常用命令示例

```sql
-- SQL语句：查询前10条记录（使用LIMIT）
SELECT * FROM users ORDER BY id DESC LIMIT 10;  -- SQLite语法 

-- 相同逻辑在SQL Server中
-- SELECT TOP 10 * FROM users ORDER BY id DESC; -- 不适用于SQLite

-- 点命令：查看表结构（仅在sqlite3命令行工具中有效）
.schema users
```

理解SQLite与标准SQL的差异，以及其特有的术语和命令，是高效使用这款轻量级数据库的关键。
