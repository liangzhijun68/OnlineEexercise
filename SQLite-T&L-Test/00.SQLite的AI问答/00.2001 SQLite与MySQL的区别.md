## 一、数据类型差异

| 对比项 | **SQLite** | **MySQL** |
| :--- | :--- | :--- |
| **类型系统** | **动态类型**（类型亲和性），可存任意类型 | **静态严格类型**，列必须匹配定义类型 |
| **自增主键** | `INTEGER PRIMARY KEY` 自动成为`ROWID`别名 | `AUTO_INCREMENT` 属性，需显式声明 |
| **字符串类型** | 仅`TEXT`（无`VARCHAR`长度限制概念） | `VARCHAR(n)`、`CHAR(n)`、`TEXT`等细粒度 |
| **日期时间** | 无原生日期类型，用`TEXT`/`INTEGER`存储 | 有`DATE`、`DATETIME`、`TIMESTAMP`等专用类型 |
| **布尔类型** | 无，用`0/1`或`INTEGER`模拟 | 有`BOOLEAN`（实际是`TINYINT(1)`） |
| **数字类型** | `INTEGER`、`REAL`（无精度指定） | `INT`、`BIGINT`、`DECIMAL(10,2)`等精确控制 |

## 二、DDL命令差异（建表改表）

| 功能 | **SQLite** | **MySQL** |
| :--- | :--- | :--- |
| `CREATE TABLE` | 支持`WITHOUT ROWID`选项 | 无此概念，默认有`_rowid`隐藏列 |
| `ALTER TABLE` | **极简**：仅支持重命名表和**添加列**（不支持删除/修改列类型） | **全功能**：支持`ADD`/`DROP`/`MODIFY`/`CHANGE`列 |
| `DROP COLUMN` | ❌ 不支持（需重建表） | ✅ 完全支持 `ALTER TABLE ... DROP COLUMN` |
| `RENAME COLUMN` | SQLite 3.25+ 支持 | ✅ 完全支持 |
| `CREATE INDEX` | 支持`CONCURRENTLY`（并发建索引，但锁表） | 支持`ALGORITHM=INPLACE`（在线DDL，不锁表） |
| `DROP INDEX` | 标准语法 | 支持`ALTER TABLE ... DROP INDEX` |

---

## 三、DML命令差异（增删改查）

| 功能 | **SQLite** | **MySQL** |
| :--- | :--- | :--- |
| **插入冲突处理** | `INSERT OR REPLACE` / `INSERT OR IGNORE` | `INSERT ... ON DUPLICATE KEY UPDATE` |
| **替换语法** | `REPLACE INTO` | `REPLACE INTO`（行为略有差异） |
| **更新联表** | 支持 `UPDATE ... FROM ... WHERE` | 支持 `UPDATE ... JOIN ... SET ...`（不同语法） |
| **删除联表** | 支持 `DELETE ... USING` | 支持 `DELETE ... JOIN` |
| **返回修改行** | ✅ `RETURNING` 子句（返回插入/更新/删除的行） | ❌ 不支持（需用`LAST_INSERT_ID()`或触发器） |
| **分页查询** | `LIMIT offset, count` 或 `LIMIT count OFFSET offset` | 同SQLite，语法一致 |
| **全文搜索** | 需启用FTS5扩展模块 | 内置`MATCH() AGAINST()`全文索引 |

---

## 四、事务与锁差异

| 对比项 | **SQLite** | **MySQL** |
| :--- | :--- | :--- |
| **事务隔离级别** | 支持`DEFERRED`/`IMMEDIATE`/`EXCLUSIVE`三种模式 | 支持`READ UNCOMMITTED`到`SERIALIZABLE`四级 |
| **默认提交行为** | **自动提交**（每个SQL语句独立事务） | 默认**自动提交**，但可`START TRANSACTION`显式开启 |
| **锁粒度** | **数据库级锁**（写操作锁整个库） | **行级锁**（InnoDB引擎，高并发） |
| **外键约束** | 默认**关闭**，需`PRAGMA foreign_keys=ON;` | 默认**开启**（InnoDB引擎） |
| **保存点** | `SAVEPOINT`完全支持 | `SAVEPOINT`完全支持 |

---

## 五、特有命令与功能

| 功能 | **SQLite（独有）** | **MySQL（独有）** |
| :--- | :--- | :--- |
| **数据库管理** | `ATTACH` / `DETACH` 附加多个数据库文件 | 无此概念，用`USE db_name`切换 |
| **状态查询** | `PRAGMA` 指令（如`PRAGMA table_info`） | `SHOW` 命令（如`SHOW TABLES`、`SHOW CREATE TABLE`） |
| **空间回收** | `VACUUM` 重建数据库文件 | `OPTIMIZE TABLE` 或 `ALTER TABLE ... ENGINE=InnoDB` |
| **存储过程/函数** | ❌ 不支持 | ✅ 完全支持（`CREATE PROCEDURE` / `FUNCTION`） |
| **触发器** | 支持`BEFORE`/`AFTER`/`INSTEAD OF` | 仅支持`BEFORE`/`AFTER`（无`INSTEAD OF`） |
| **视图** | **只读**（除非创建`INSTEAD OF`触发器） | 支持**可更新视图**（满足条件时） |
| **JSON支持** | 有限（需扩展，或存为TEXT） | ✅ 原生JSON类型和函数（`JSON_EXTRACT`等） |

## 六、命令行工具对比

| 功能 | **SQLite（sqlite3）** | **MySQL（mysql客户端）** |
| :--- | :--- | :--- |
| **查看表结构** | `.schema table_name` | `DESC table_name;` 或 `SHOW CREATE TABLE;` |
| **列出所有表** | `.tables` | `SHOW TABLES;` |
| **导出数据** | `.dump > file.sql` | `mysqldump` 独立工具 |
| **导入数据** | `.read file.sql` | `SOURCE file.sql;` |
| **切换数据库** | `.open other.db` | `USE db_name;` |
| **查看当前库** | `.databases` | `SELECT DATABASE();` |
| **执行系统命令** | `.shell command` | `\! command` |
| **退出** | `.quit` 或 `.exit` | `quit;` 或 `exit;` |

## 七、核心术语对照

| 术语 | **SQLite** | **MySQL** |
| :--- | :--- | :--- |
| **数据库文件** | 单个`.db`文件即完整数据库 | 多个文件（`.frm`、`.ibd`等）存储在数据目录 |
| **ROWID** | 每个表默认隐藏的**自增64位整数** | 无此概念，用`AUTO_INCREMENT`显式列 |
| **类型亲和性** | 列有倾向类型但不强制 | 列强制类型，插入错误类型会报错或转换 |
| **并发控制** | 写操作**串行化**（适合低并发） | MVCC多版本并发控制（适合高并发） |
| **存储引擎** | 单一引擎（无选择） | 多引擎（InnoDB默认，MyISAM等可选） |
| **大小写敏感** | Unix系**区分**表名大小写，Windows不区分 | 取决于操作系统和`lower_case_table_names`设置 |

## 总结建议

- **选SQLite**：单机应用、移动端、嵌入式、原型开发、读写并发低
- **选MySQL**：Web服务、高并发、复杂查询、需要存储过程/函数、多用户写入
