你说得对，“主键”（`PRIMARY KEY`）和“自增”（`AUTOINCREMENT`）这类概念，严格来说不属于“命令”（Command）。

在SQLite中，它们属于**“语法元素”（Syntax Elements）**，具体可细分为**“列约束”（Column Constraint）**和**“关键字”（Keyword）**。它们用于在`CREATE TABLE`等语句中定义表结构。

以下是一份更详细的分类中英文对照表，帮助你系统记忆。

### 🔑 列约束 (Column Constraints)
用于确保数据的完整性，在`CREATE TABLE`中定义。

| 英文 (English) | 中文 (Chinese) | 简要说明 |
| :--- | :--- | :--- |
| **`PRIMARY KEY`** | **主键** | 唯一标识表中的每一行。 |
| **`AUTOINCREMENT`** | **自增** | 关键字，与`INTEGER PRIMARY KEY`连用，确保键值在表生命周期内唯一。 |
| **`FOREIGN KEY`** | **外键** | 维护表与表之间的引用完整性。 |
| **`UNIQUE`** | **唯一** | 确保列中的所有值各不相同。 |
| **`NOT NULL`** | **非空** | 确保列不能有`NULL`值。 |
| **`DEFAULT`** | **默认值** | 当未指定值时，为该列提供默认值。 |
| **`CHECK`** | **检查** | 确保列中所有值满足特定条件。 |
| **`COLLATE`** | **排序规则** | 定义用于比较文本的排序规则。 |

### 🏷️ 数据类型 (Data Types) / 存储类 (Storage Classes)
SQLite采用动态类型系统，值的类型与值本身关联。

| 英文 (English) | 中文 (Chinese) | 简要说明 |
| :--- | :--- | :--- |
| **`NULL`** | **空值** | 表示缺失或未知的信息。 |
| **`INTEGER`** | **整数** | 有符号整数，根据大小灵活存储。 |
| **`REAL`** | **浮点数** | 8字节IEEE浮点数。 |
| **`TEXT`** | **文本** | 文本字符串，使用数据库编码存储。 |
| **`BLOB`** | **二进制大对象** | 数据按输入时的原样存储。 |

### 🔍 特殊列名 (Special Column Names)
每个表（`WITHOUT ROWID`除外）都有一个自增的64位有符号整数`ROWID`。

| 英文 (English) | 中文 (Chinese) | 简要说明 |
| :--- | :--- | :--- |
| **`ROWID`** | 行ID | 访问`ROWID`的三种标准方式。 |
| **`_ROWID_`** | 行ID | 访问`ROWID`的三种标准方式。 |
| **`OID`** | 对象ID | 访问`ROWID`的三种标准方式。 |
| **`INTEGER PRIMARY KEY`** | 整数主键 | 声明后，该列会成为`ROWID`的别名。 |

### 🔧 特殊指令 (Pragmas)
`PRAGMA`语句是SQLite特有的指令，用于修改或查询数据库的运行方式。

| 英文 (English) | 中文 (Chinese) | 简要说明 |
| :--- | :--- | :--- |
| **`PRAGMA foreign_keys`** | 外键约束开关 | 启用(`=ON`)或禁用(`=OFF`)外键约束检查。 |
